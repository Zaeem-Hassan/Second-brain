# Tool Use

## Summary

Tool use is the mechanism that lets an agent extend beyond pure text generation by calling external systems such as search, code execution, APIs, databases, or file operations. It is one of the main capabilities that turns a model into a task-performing system.

## Key Points

- Tool use gives agents access to fresh information and real-world actions.
- Reliable tool use depends on schema quality, validation, error handling, and retry strategy.
- Tools increase capability, but also increase failure modes and operational risk.
- MCP-style interfaces matter because they standardize how models discover and invoke tools.
